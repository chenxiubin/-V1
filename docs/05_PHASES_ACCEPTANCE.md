# 分阶段开发路线与验收门禁

## 总规则

- 一次只执行一个Phase。
- 当前Phase的P0验收未通过，不进入下一Phase。
- 每阶段结束必须更新PROJECT_HANDOFF.md。
- 不删除、跳过测试或降低断言绕过失败。
- 不使用Mock冒充真实模型验证。

## 全局中文界面验收

- AC-001 应用名称显示为“台历智能场景规划平台”。
- AC-002 所有用户可见导航、标题、按钮、占位文字、状态、错误和报告使用简体中文。
- AC-003 页面不存在无关的通用AI聊天、角色预设、头脑风暴、提示词工程或文案润色模块。
- AC-004 必要技术缩写可以保留，但不出现整块英文界面或中英混排菜单。

AC-001～AC-004适用于所有Phase。任一不通过，不得声明当前阶段完成。

## Phase 0｜基线与项目结构

目标：确认React/Node项目结构、脚本、预览、测试和Secrets边界。

完成条件：

- 记录目录、入口、状态管理和持久化。
- 运行现有测试、类型检查、Lint和构建。
- 创建或更新PROJECT_HANDOFF.md。
- 不实现业务功能。

## Phase 1｜领域模型、Schema、状态机

目标：建立ProductAsset、ProductProfile、SceneRecipe、PromptDocument、MatchReport、RecipePatch和SeriesProject。

验收：

- AC-101 非法模型数据不能进入Store。
- AC-102 Recipe可创建新版本和恢复旧版本。
- AC-103 页面刷新后文本项目可恢复。
- AC-104 Mock和真实Adapter明确区分。
- AC-105 SceneRecipe是唯一事实源。

## Phase 2｜产品导入与多模态分析

验收：

- AC-201 支持透明PNG预览、尺寸和Alpha状态。
- AC-202 无产品不能分析。
- AC-203 分析失败保留产品并可重试。
- AC-204 ProductProfile必须通过Schema。
- AC-205 低置信度字段明确显示。
- AC-206 替换产品使旧分析和Recipe过期。
- AC-207 产品不会被镜像、重绘或透视变形。

## Phase 3｜引导问题与场景方向

验收：

- AC-301 每轮不超过5个问题。
- AC-302 不重复询问高置信度事实。
- AC-303 返回3个合法SceneDirection。
- AC-304 三个方向存在明确差异。
- AC-305 推荐方向有产品相关理由。
- AC-306 换一批不会重新创建ProductProfile。

## Phase 4｜Recipe、提示词与JSON

验收：

- AC-401 相同Recipe输出逐字符一致。
- AC-402 完整提示词按固定6段编译。
- AC-403 完整JSON可解析。
- AC-404 单独复制一级对象仍是合法JSON。
- AC-405 复制成功有反馈。
- AC-406 输出不含Key、Base64、对象URL和内部路径。
- AC-407 输出明确为空场景。
- AC-408 不存在平台内生图入口。

## Phase 5｜场景导回与产品预览

验收：

- AC-501 支持剪贴板粘贴。
- AC-502 支持文件上传备用。
- AC-503 场景底层、真实产品上层。
- AC-504 产品初始位置来自Recipe。
- AC-505 产品像素未改变。
- AC-506 明确标注未做光影融合。
- AC-507 替换场景使旧报告过期。

## Phase 6｜匹配分析与补丁

验收：

- AC-601 无场景结果不能分析。
- AC-602 报告包含严重度、证据、建议和置信度。
- AC-603 报告不能只有总分。
- AC-604 不确定时返回uncertain。
- AC-605 Patch只能使用允许路径。
- AC-606 用户确认前不能应用Patch。
- AC-607 应用Patch创建新Recipe版本。
- AC-608 无关字段保持一致。
- AC-609 用户可以恢复旧版本。
- AC-610 模型不能自动标记通过。

## Phase 7｜通过与系列母版

验收：

- AC-701 没有真实场景不能标记通过。
- AC-702 只有已通过图片可创建系列。
- AC-703 系列母版保存实际场景和确切版本。
- AC-704 可选择同空间或同风格。
- AC-705 系列不固定图片数量。

## Phase 8｜后续产品自动规划

验收：

- AC-801 新产品创建独立ProductProfile。
- AC-802 系统参考已覆盖用途避免机械重复。
- AC-803 同空间锁不会被单张规划覆盖。
- AC-804 同风格模式允许新SceneGroup。
- AC-805 不兼容时显示真实风险。
- AC-806 单张修正不修改母版。
- AC-807 更新系列标准需要用户确认并创建新版本。
- AC-808 系列报告包含产品匹配和系列一致性。

## Phase 9｜稳定性、安全和回归

验收：

- AC-901 刷新后项目、Recipe、Prompt和报告可恢复。
- AC-902 大图片不长期以Base64写入localStorage。
- AC-903 GEMINI_API_KEY不出现在客户端、日志和复制文本。
- AC-904 模型超时、网络失败和拒绝可恢复。
- AC-905 全流程E2E通过。
- AC-906 类型检查、Lint和生产构建通过。
- AC-907 已知未完成功能明确记录，没有伪装完成。

## 每阶段结束必须执行

根据项目实际脚本运行：

```text
单元测试
集成/E2E测试
类型检查
Lint
生产构建
```

如果项目没有某类脚本，在交接中写“项目未提供”，不得伪造结果。

## 手工E2E主流程

```text
上传透明产品
→ 产品分析
→ 回答引导问题
→ 选择场景方向
→ 复制提示词和JSON
→ 粘贴外部空场景
→ 产品叠加预览
→ 匹配分析
→ 采纳修正
→ 生成V2提示词
→ 导入新场景
→ 用户标记通过
→ 创建系列
→ 导入下一张产品
```
