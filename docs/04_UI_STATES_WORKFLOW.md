# UI状态、页面与交互流程

## 1. 设计原则

- 单一智能模式。
- 全站用户界面使用简体中文，不允许生成英文工作台或中英混排界面。
- 应用标题固定为“台历智能场景规划平台”。
- 用户不操作专业摄影参数。
- AI推荐必须可理解、可更换、可撤销。
- 每个页面只有一个主要下一步。
- 所有异步功能包含加载、失败、重试。
- 预览明确标注未做光影融合。

### 1.1 中文界面规则

- 所有导航、页签、按钮、卡片标题、输入框占位、提示、错误、加载、空状态和分析报告使用简体中文。
- 禁止使用“AI Presets Chat”“Brainstorm Map”“Prompt Engineer”“Refine Copy”“Active Agent Persona”等无关英文模块。
- 必要技术词允许保留JSON、PNG、API、AI、Nano Banana，但同一控件中的操作说明必须是中文。
- 日期、时间和版本状态按中文阅读习惯展示。
- 复制成功、分析失败、模型超时等反馈必须有中文文案。

## 2. 状态机

```text
EMPTY
→ PRODUCT_IMPORTED
→ ANALYZING_PRODUCT
→ PRODUCT_REVIEW
→ GUIDED_QUESTIONS
→ DIRECTION_SELECTION
→ RECIPE_READY
→ AWAITING_EXTERNAL_GENERATION
→ PREVIEW_IMPORTED
→ ANALYZING_MATCH
→ NEEDS_REVISION / APPROVED
→ SERIES_ACTIVE（可选）
```

禁止跳过必要状态。例如没有外部场景图不能进入匹配分析或标记通过。

## 3. 页面结构

建议三栏：

- 左侧：步骤、产品和系列图片缩略图。
- 中间：当前步骤内容或产品叠加预览。
- 右侧：AI摘要、方向、提示词、JSON、报告和修正。

## 4. 状态详细要求

### EMPTY

- 产品拖放区。
- 选择或粘贴透明PNG。
- 不显示虚假分析结果。

### PRODUCT_IMPORTED

- 棋盘格产品预览。
- 尺寸、Alpha状态。
- 开始智能分析。
- 替换产品。

### ANALYZING_PRODUCT

- 显示真实阶段文案，不显示伪百分比。
- 支持失败重试。
- 失败不清除产品。

### PRODUCT_REVIEW

- 通俗展示类型、视角、材质、现有光线、推荐背景关系。
- 重点显示低置信度项。
- 允许修正明显错误，不展示复杂参数面板。

### GUIDED_QUESTIONS

- 一次一个问题或一组相关问题。
- 2～3个选项，推荐项置前。
- 支持由AI决定。
- 每轮最多5题。

### DIRECTION_SELECTION

每个方向卡片显示：

- 名称和效果摘要。
- 推荐理由。
- 空间和台面。
- 色调和光线。
- 产品预留和文案位置。
- 风险。

操作：选择、换一批、返回修改答案。

### RECIPE_READY

显示：

- 当前方向。
- Recipe版本。
- 完整提示词。
- 6个分段提示词。
- 完整JSON。
- 一级JSON对象。

操作：

- 复制完整提示词。
- 单段复制。
- 复制完整JSON。
- 单对象复制。
- 进入外部生成说明。

禁止显示“平台内生成图片”按钮。

### AWAITING_EXTERNAL_GENERATION

显示：

- 需要携带的图片角色。
- 需要复制的文本。
- 空场景输出要求。
- 粘贴场景图。
- 文件上传备用。

### PREVIEW_IMPORTED

- 场景在底层，原始产品在上层。
- 产品位置和占比来自Recipe。
- 不允许镜像和透视变形产品。
- 支持显示/隐藏、适应视图、原始比例和替换场景。
- 标注“构图匹配预览，未做光影融合”。

### ANALYZING_MATCH

- 显示分析输入摘要。
- 失败不删除场景和产品。

### NEEDS_REVISION

按类别显示：

- 透视与接触。
- 构图与留白。
- 光线。
- 色彩和主体突出。
- 场景语义。
- 系列一致性。

每个问题显示严重度、证据、建议和置信度。

操作：单项采纳、全部采纳、忽略、换方向。

### APPROVED

- 只能用户确认。
- 显示最终场景、叠加预览、Recipe和Prompt版本。
- 可保持独立图片或创建系列。

### SERIES_ACTIVE

- 显示系列母版和系列风格摘要。
- 显示场景组和已有图片。
- 提供“导入下一张产品”。
- 不显示固定总张数目标。

## 5. 系列下一张交互

新产品分析完成后，系统显示AI规划和兼容性理由。用户只选择：

- 接受AI规划。
- 换一个方向。
- 必须同一空间。
- 只保持同一风格。

若新产品与同空间光线或视角明显冲突，必须警告，不能假装一定兼容。

## 6. 快捷纠错

- 太复杂。
- 太空。
- 产品不突出。
- 透视不自然。
- 光线不一致。
- 色调不合适。
- 不像同系列。
- 不像同空间。
- 只换构图。
- 换空间但保留风格。

快捷纠错先转换为RecipePatch，再由用户确认。

## 7. 错误恢复

- 图片格式错误：不创建资产。
- 模型失败：保留产品、答案和当前页面。
- Schema失败：不进入下一步。
- 剪贴板失败：显示上传入口。
- 产品替换：旧结果标记过期。
- 场景替换：旧MatchReport标记过期。
- 系列参考图缺失：禁止生成严格同空间提示词。
- 持久化失败：明确告知可能未保存。

## 8. 禁止交互

- 不自动标记通过。
- 不自动创建系列。
- 不显示大量专业滑块。
- 不用静态卡片冒充真实AI结果。
- 不允许水平翻转产品。
- 不把透明度降低当成景深虚化。
- 不让单张操作修改系列母版。
