# 台历智能场景规划平台 V1｜产品需求与范围

## 1. 产品定位

用户上传一张已经抠图精修的真实台历透明PNG，平台通过多模态分析和少量引导式问题，自动规划适合产品的空场景、构图、光线和风格，生成可复制的Nano Banana中文提示词与JSON文本。用户在外部页面生成空场景后，把图片粘贴或上传回平台；平台用真实产品叠加预览，并通过多模态分析修正提示词。第一张合格结果可以建立系列，后续产品图导入后由系统自动规划同风格图片。

## 2. 核心用户流程

```text
透明产品PNG
→ 产品多模态分析
→ 用户确认低置信度信息
→ 2～5个引导式问题
→ 3个场景方向
→ 用户选择一个方向
→ 生成SceneRecipe
→ 编译提示词和JSON
→ 用户复制到外部Nano Banana页面
→ 生成空场景
→ 场景图片粘贴回平台
→ 原始产品叠加预览
→ 多模态匹配分析
→ 采纳修正并生成新版本提示词
→ 用户标记通过
→ 可选创建系列
```

## 3. V1必须包含

- 单一智能模式。
- 全部面向用户的界面使用简体中文。
- 产品PNG上传、粘贴、预览、替换。
- 产品视角、材质、颜色、现有光线和置信度分析。
- 引导式问题。
- 3个场景方向及推荐理由。
- SceneRecipe结构化规划。
- 完整提示词和6个分段提示词。
- 完整JSON和一级对象JSON。
- 一键复制和单段复制。
- 外部场景图粘贴/上传。
- 产品叠加预览。
- 匹配分析、受限补丁、版本历史。
- 第一张通过后创建系列。
- 后续产品动态加入系列。
- 同空间和同风格两种继承策略。

## 4. V1明确不做

- 不提供专业模式。
- 不提供复杂摄影参数面板。
- 不直接调用Nano Banana生图API。
- 不接ComfyUI或RunningHub。
- 不生成或下载生成包、ZIP。
- 不做最终光影融合和接触阴影。
- 不重绘或修改产品。
- 不自动批量生成整套图片。
- 不固定系列图片数量。
- 不做登录、Firebase、团队协作和云端素材库。
- 不承诺生成模型像素级遵守构图。
- 不开发通用AI聊天、角色预设、头脑风暴、提示词工程或文案润色工作台。

## 4.1 界面语言与产品命名

- 应用名称固定为“台历智能场景规划平台”。
- 导航、标题、按钮、输入框占位文字、说明、加载状态、空状态、错误信息、分析报告、复制反馈和版本状态全部使用简体中文。
- 面向用户的界面不得出现整段英文、英文菜单或中英混排。
- 必要技术缩写可以保留，例如JSON、PNG、API、AI、Nano Banana，但必须放在中文语境中。
- 内部代码标识、类型名和接口路径可以使用英文，不属于界面语言限制。
- 模型返回的英文说明在展示前必须转换为中文结构化字段或中文文案。

## 5. 功能需求

### FR-01 产品导入

- 优先支持透明PNG，兼容JPEG/WebP并提示透明度风险。
- 显示尺寸、透明状态和产品预览。
- 替换产品后旧分析、旧Recipe和旧报告必须标记过期。
- 产品不得自动水平翻转、透视拉伸或重绘。

### FR-02 产品分析

至少输出：

- 产品类型和支架类型。
- 单品、组合或包装关系。
- 主体边界框和底部接触区域。
- 正面、左前侧、右前侧、轻俯拍或高俯拍。
- 顶部和侧面可见程度。
- 材质和反光程度。
- 主色和边缘明暗。
- 已有主亮面方向、色温、光线软硬和对比度。
- 不确定字段、原因和置信度。

无法判断时返回`unknown`，不得伪造精确角度。

### FR-03 引导式问题

- 每轮最多5个问题。
- 每题提供2～3个互斥选项。
- 推荐项置前并说明一至两句理由。
- 只询问用户偏好或低置信度内容。
- 重点询问图片用途、风格方向、背景丰富度、文案留白和系列继承。

### FR-04 场景方向

- 默认提供3个方向。
- 三个方向在空间、构图或审美上有明确差异。
- 至少一个为稳妥高成功率方案。
- 每个方向包含空间、台面、色调、光线、装饰、构图、推荐理由和风险。
- 用户可以选择或换一批；换一批不重新分析产品。

### FR-05 SceneRecipe

- 选中方向后生成结构化SceneRecipe。
- 所有提示词、JSON、预览位置和分析基准来自同一个Recipe版本。
- 智能模式只显示通俗摘要，不暴露专业数值编辑。

### FR-06 提示词和JSON

固定6段：

1. 任务与参考关系。
2. 产品匹配依据。
3. 场景与风格。
4. 镜头与构图。
5. 光线与装饰。
6. 输出限制。

要求：

- 完整提示词一键复制。
- 每段单独复制。
- 完整JSON一键复制。
- 每个一级对象单独复制后仍是合法JSON。
- 相同Recipe版本必须生成逐字符一致文本。
- 输出不包含Key、Base64、对象URL或内部路径。

### FR-07 外部生图

- 平台只给出使用清单，不调用生图API。
- 输出目标为空场景，不生成产品、人物、手部、文字、标识和水印。
- 严格同空间系列提醒用户携带同一系列参考图。

### FR-08 场景导回与预览

- 支持剪贴板粘贴，文件上传作为备用。
- 场景图在底层，真实透明产品在上层。
- 产品初始位置和占比来自SceneRecipe。
- 只做叠加预览，不做光影融合。
- 支持显示/隐藏产品、适应视图、替换场景。

### FR-09 匹配分析

分析：

- 产品与台面透视和接触关系。
- 产品位置、占比和文案留白。
- 产品已有光线与场景光源。
- 产品与背景的明暗和色彩分离。
- 场景语义是否合理。
- 装饰是否抢夺或遮挡主体。
- 系列模式下的色彩、材质、光线、摄影质感和空间一致性。

报告必须给出问题类型、严重度、证据、建议字段和置信度，不能只给总分。

### FR-10 修正闭环

- 分析生成受限RecipePatch。
- 用户可以单项采纳、全部采纳或忽略。
- 用户确认后创建新Recipe版本和Prompt版本。
- 只修改有关字段。
- 单张补丁不得修改系列母版。

### FR-11 通过状态

- 状态：草稿、需要修正、已通过。
- 必须有真实外部场景图才能标记通过。
- 只能由用户确认通过，模型不能自动通过。

### FR-12 创建系列

- 只有已通过图片可以创建系列。
- 保存实际场景图、ProductProfile、SceneRecipe、Prompt和MatchReport快照。
- 用户选择严格同空间或同风格扩展。
- 第一张实际场景图是系列视觉母版，第一张提示词不是视觉母版。

### FR-13 后续产品规划

- 每个新产品重新分析。
- 系统比较新产品与系列母版的视角、材质、光线和构图适配性。
- 参考系列已覆盖用途，避免机械重复。
- 自动推荐同空间、同风格新空间或提示不兼容。
- 用户只选择接受、换方向、必须同空间或只保持同风格。

## 6. 快捷反馈

- 场景太复杂。
- 场景太空。
- 产品不突出。
- 透视不自然。
- 光线不一致。
- 色调不合适。
- 不像同系列。
- 不像同空间。
- 保持空间只换构图。
- 换空间但保留风格。

快捷反馈必须映射为结构化补丁，不直接追加到提示词末尾。

## 7. 非功能需求

- 页面刷新后可恢复项目、Recipe、Prompt和报告。
- 图片使用IndexedDB或现有资产层，不长期以Base64存入localStorage。
- 所有异步功能都有加载、失败和重试。
- API Key只存在服务端Secret。
- 错误日志不保存完整图片Base64。
- 复制操作不等待模型调用。
- 模型分析不得虚构合格结果。
